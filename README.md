# CHEREPOVETS Moderation v49

Next.js-сборка для рабочего сайта модерации.

Этап 1:
- сайт отдаётся через Next route handler;
- текущий рабочий HTML сохранён в `legacy/index.html`;
- ассеты и Supabase config лежат в `public`;
- функционал v41 не вырезается.

Этап 2:
- добавлены отдельные URL для ключевых разделов:
  `/profile`, `/reports`, `/inactives`, `/reports-table`, `/shop`, `/leaderboard`, `/leadership`, `/creator`, `/guide`, `/general`;
- route handler подставляет нужный режим в старое приложение через безопасный bridge script;
- добавлен `/api/navigation` со списком страниц.

Этап 3:
- добавлен первый настоящий Next-раздел `/native/profile`;
- старый рабочий `/profile` оставлен без замены;
- native-профиль читает Supabase-сессию, `user_stats`, `reports`, игровой XP, реальный XP и роли;
- добавлена отдельная мобильная тема без зелёной палитры.

Этап 4:
- добавлен настоящий Next-раздел `/native/inactives`;
- старый рабочий `/inactives` оставлен как резерв;
- native-форма пишет заявки в старый формат `reports / INACTIVE_REQ`;
- сохраняется совместимость с таблицей отчётов: `Ник: ... | С: ... | По: ... | Причина: ...`.

Этап 5:
- добавлен настоящий Next-раздел `/native/reports`;
- старый рабочий `/reports` оставлен как резерв;
- native-форма пишет отчёт в старый формат `reports`;
- сохраняется совместимость с таблицей отчётов: `Ник: ... | Дата: ... | Работа: ... | Тип сдачи: ... | JSON: ...`;
- поддержаны ссылки и один загружаемый скриншот с браузерным сжатием.

Этап 6:
- добавлен настоящий Next-раздел `/native/review`;
- руководитель может проверять отчёты без legacy-экрана;
- действия пишут напрямую в `reports.status` и `reports.xp`;
- статусы и XP совместимы со старой таблицей: `Норма=15`, `Перенорма=30`, `Натяг=7`, `Герой дня=60`, `Не засчитано=0`.

Этап 7:
- добавлен настоящий Next-раздел `/native/table`;
- старая рабочая таблица `/reports-table` оставлена как резерв;
- таблица строится по модераторам из `reports` с `email=USER_ROLE` и `status=moderator`;
- отчёты и одобренные неактивы подтягиваются из старой таблицы `reports`;
- руководитель может менять статус существующего отчёта, XP обновляется теми же значениями, что и в проверке отчётов.

Этап 8:
- исправлен TypeScript-конфликт `window.supabase`, который ломал Vercel build;
- добавлен общий native-shell для `/native/*`;
- добавлена главная native-страница `/native`;
- native-разделы получили единую верхнюю навигацию без удаления legacy-резерва.

Следующие этапы:
- перенести Supabase-логику в отдельные модули;
- перенести панель создателя и управление ролями в native;
- постепенно заменить legacy-разделы native-страницами без потери старых функций.
