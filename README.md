# CHEREPOVETS Moderation v58

Полностью переписанный сайт команды модерации: современный Next.js (App Router), светлая и тёмная темы, анимации, без legacy-кода.

## Что внутри

- **Авторизация** — вход и регистрация через Supabase Auth (та же база, все аккаунты сохранены).
- **Дашборд** — сводка XP, уровня, отчётов и быстрые действия.
- **Отчёты** — подача отчётов (в старом формате таблицы `reports`), список своих отчётов со статусами.
- **Проверка** — панель руководства: одобрение/отклонение отчётов с начислением XP (через серверный API).
- **Неактивы** — заявки на неактив и их статусы.
- **Таблица** — общая таблица всех отчётов с фильтрами.
- **Лидерборд** — рейтинг модераторов по XP.
- **Профиль** — ник, ранг, статистика, редактирование.
- **Магазины** — за XP модерации, за AP и ФСБ-баллы; покупки логируются для руководства.
- **Игры** — 10 мини-игр за XP: рулетка, монетка, сейфы, краш, кейсы, кости, мины, hi-lo, колесо, напёрстки.
- **Обучение** — тренажёры модератора/саппорта/жалоб, еженедельные квизы, квесты.
- **Инструкция** — руководство для модераторов (редактируется руководством).
- **Админ-панель** — управление пользователями (роли, ручное начисление XP), логи покупок, редактор магазинов и инструкций.

## Установка

1. Установите зависимости: `npm install`
2. Создайте `.env.local` со значениями:
   - `NEXT_PUBLIC_SUPABASE_URL` — URL проекта Supabase
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY` — публичный anon-ключ
   - `SUPABASE_SERVICE_ROLE_KEY` — секретный service_role ключ (для серверных API)
   - `VK_GROUP_TOKEN`, `VK_API_VERSION` — (опционально) для VK-уведомлений
3. Запуск разработки: `npm run dev`
4. Сборка: `npm run build && npm start`

## Технологии

- Next.js 15 (App Router) + React 19 + TypeScript
- Tailwind CSS v4 + Radix UI + framer-motion
- Supabase (Auth + Postgres), существующая схема таблиц сохранена:
  `reports` (включая служебные KV-записи), `user_stats`, `admin_logs`, `vk_links`, `moderator_careers`

## Безопасность

- Вердикты по отчётам и запись профиля выполняются на сервере через `SUPABASE_SERVICE_ROLE_KEY` (не попадает в браузер).
- Никакого `innerHTML` — только React-рендеринг.
- Роли (модератор / руководство / создатель) читаются из тех же записей `ADMIN_ROLE`, что и раньше.
